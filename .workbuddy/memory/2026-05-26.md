# 2026-05-26 工作日志

## 大屏控制服务打包方案

### 创建的文件
- `build-exe.bat` - 打包脚本，使用 pkg 5.8.1 将 server.js 打包为 exe
- `配置.bat` - 交互式配置向导，读写 config.env
- `启动.bat` - 读取 config.env 并启动 大屏控制服务.exe
- `设置开机自启动.bat` - 写入注册表 HKCU\...\Run 实现开机自启（无需管理员）

### 关键技术点
- pkg 打包原生 .node 模块有限制，robotjs.node 需与 exe 同目录（不能打进 exe）
- server.js 头部加了兼容逻辑：exe 运行时从 `path.dirname(process.execPath)` 加载 robotjs.node；开发时 fallback 到 node_modules
- build-exe.bat 打包后自动执行 `copy robotjs.node -> dist\robotjs.node`
- dist 目录结构：大屏控制服务.exe + robotjs.node + 配置.bat + 启动.bat + 设置开机自启动.bat + config.env（运行时生成）
- pkg 首次打包需下载 node18 基础包（约50MB），需要网络

### 依赖
- 全局安装 pkg：`npm install -g pkg`（已安装 5.8.1）
- @jitsi/robotjs（robotjs 预编译替代，无需 VS 编译环境）

### 修复 pkg 打包后 robotjs 原生模块加载失败
- 错误: `No native build was found` — pkg 把 .node 打进虚拟文件系统，原生模块无法加载
- 根因: `node-gyp-build` 在 pkg 虚拟快照中找不到 prebuilds/.node
- 方案: 利用 node-gyp-build 的 `nearby` fallback 机制——在 exe 同级目录放置 `prebuilds/win32-x64/@jitsi+robotjs.node`，运行时自动发现
- build.js 新增 Step 3.5: 自动从 node_modules 复制 prebuilds 目录到 dist
- server.js 无需修改
- dist 最终结构: ControlTV.exe + prebuilds/win32-x64/@jitsi+robotjs.node + *.bat + config.txt

### 创建 remote.html 响应式遥控面板
- 纯 HTML+CSS+JS 单文件，桌面/手机响应式布局
- 桌面端：左侧边栏(连接)+右侧 tab 切换(Quick/Nav/Keyboard/Mouse/Open)
- 手机端：底部 tab bar 切换，顶部可显示连接状态栏（后续可加）
- 5个功能面板：
  - Quick: 浏览器控制(刷新/关闭标签/切换标签/F5/F11) + 系统控制(Alt+F4/Win+Tab/Esc/音量)
  - Navigation: 方向键 D-Pad + 编辑键(Esc/Tab/Enter/Space/Backspace/Del/Ins/Home/End/PgUp/PgDn)
  - Keyboard: 完整虚拟键盘(F1-F12/数字行/QWERTY/符号)，支持所有字母/数字/符号按键
  - Mouse: 触控板滑动 + 左键/右键点击，以1920x1080比例映射
  - Open: URL/本地文件路径输入，发送open_web JSON指令
- WebSocket 连接管理，心跳保活，Toast 反馈
- IP 地址 localStorage 记忆
- 深色主题，适配移动端触控
